What is TypeScript?

  • Typed superset of JavaScript.
  • An open-source programming language.
  • Maintained by Microsoft since they created it in 2012.
  • Transcompiles to JavaScript.
  • May be used for both: client-side and server-side.

History

  • TypeScript 0.8
    October 2012, after two years of internal development at Microsoft.
  • TypeScript 1.0
    In 2014 on Microsoft's Build developer conference.
  • TypeScript 2.0
    22 September 2016.
  • TypeScript 3.0
    30 July 2018.

Compatibility with JS

TypeScript is a strict superset of ECMAScript 2015.
ECMAScript 2015 is a superset of ECMAScript 5 (JavaScript).

As such, a JS program is also a valid TS program.
A TS program can seamlessly consume JS.

The Benefits of Using TS

  • Static typing
    => Code is more predictable, easier to debug

  • Modules, namespaces and strong OOP support
    => Easier to organize the code

  • Compilation step to JS
    => Early errors catching

Compiler

The TypeScript compiler, named tsc, is written in TS.

Can then be executed in any JS engine.

The compiler package comes bundled with a script host that can execute the compiler.

It is also available as a Node.js package that uses Node.js as a host.

Installing TS

Open the terminal and type 

npm install -g typescript
Verify it by running the command 
tsc -v
which will display the version of the TS installed

Writing some code

1. Open IDE/Text Editor
2. Create a file main.ts
3. Write a few lines of code: 

let a = 5;
let b = 5;
let c = a + b;
console.log(c);
4. The next step is to compile our TS into JS

Compiling

The compilation process can be done in a number of different ways:

  • In the terminal using tsc.
  • Directly in IDEs and text editors.
  • Using automated task runners such as gulp.

Compilation commands

Command to generate filename.js 

tsc filename.ts
To compile multiple files at once: 
tsc main.ts worker.ts
tsc *.ts

Let's compile


In the terminal at the file location run the command: 

tsc main.ts

Advanced compiling

The --watch option can be used to automatically compile a TS file when changes are made: 

tsc main.ts –watch

Also a tsconfig.json file can be created to hold various build settings.

Data types


  • Typing ensures reliability and scalability.
  • Helps in hunting down bugs and errors.
  • Helps to properly document our code.

Data types example

TS: 

let burger: string = 'hamburger';
let calories: number = 300;
function speak(food: string, energy: number): void {
    console.log(`Our ${food} has ${energy} calories.`);
}
speak(burger, calories);
JS: 
let burger = 'hamburger';
let calories = 300;
function speak(food, energy) {
    console.log(`Our ${food} has ${energy} calories.`);
}
speak(burger, calories);

Data types


  • Any type
  • Built-in types
  • User-defined types

Any type

The any data type is the superset of all the data types in TypeScript. Giving any variable the type of any is equivalent to opting out of type checking for a variable. 

let myVariable: any = 'This is a string';

Built-in types

These are the types which are built in TypeScript. They include number, string, boolean, void, null and undefined. 

let num: number = 5;
let name: string = 'Alex';
let isPresent: boolean = true;

User-defined types

The user-defined types include enum, class, interface, array and tuple.

OOP


TypeScript supports all the features of object-oriented programming, such as classes and interfaces.

Class

Class - the template of objects.
The class keyword can be used to declare one: 

class Menu {
  items: Array<string>
  pages: number;

  constructor(itemList: Array<string>, totalPages: number) {
    this.items = itemList;
    this.pages = totalPages;
  }

  list(): void {
    console.log(`Our menu for today: ${this.items.join(', ')}`);
  }
}
const sundayMenu = new Menu(["pancakes", "waffles", "juice"], 1);
sundayMenu.list();

Interface

An interface - a syntactical contract to which an object should conform.
To create an interface, we use the keyword interface: 

interface Food {
  name: string;
  calories: number;
}

function speak(food: Food): void {
  console.log(`Our ${food.name} has ${food.calories} calories.`);
}

const iceCream: Food = {
  name: "ice cream",
  calories: 200
}
speak(iceCream);
